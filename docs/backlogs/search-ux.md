# 探す体験の改善（ボトムシート）

## ステータス
📋 未着手

## 背景・課題

現在「食材を探す」ボタンから食材検索のみに遷移できる。
レシピが増えると探し方も多様になるため、複数の探し方を統一した入り口から提供したい。

また、お気に入り・調理回数など新機能が加わるとアクセス先が分散してしまうため、「探す」ハブとして一本化する。

## 方針

「食材を探す」ボタンを「探す」ボタンに変更し、タップするとボトムシートが開く。
ボトムシート内に複数の探し方をカード/ボタンで一覧表示する。

### なぜチャットボットではなくボトムシートか

チャットボット形式だと毎回1ステップ余分にかかる。
ボトムシートなら選択肢を一覧で見て即タップできる。
将来的に自由入力（「15分でできるもの」など）に対応する場合は、ボトムシート内にテキスト入力欄を追加する形で拡張できる。

## 実装アイデア

### ボトムシートの選択肢

```
┌──────────────────────────┐
│ ⭐ お気に入り             │
│ 🔁 よく作るレシピ         │
│ 🕐 最近登録したレシピ     │
│ 🥦 食材で探す            │
└──────────────────────────┘
```

### 技術

- shadcn/ui の `Sheet` コンポーネント（`side="bottom"`）を使用
- LIFF アプリ（Next.js）内での実装なので制約なし
